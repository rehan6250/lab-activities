Web Development - Lecture 1

Introduction to Web Terminologies, HTTP, Web Generations, and Standards & Constraints

1. Overview of Web Terminologies

  • Website: A collection of web pages accessible through the internet.
  • Web Page: A single document on the web, typically written in HTML and viewed in a browser.
  • URL (Uniform Resource Locator): The address used to access a web page.
  • Web Server: A computer that stores and delivers web pages to users.
  • Browser: Software (like Chrome, Firefox) that accesses and displays web content.
  • Frontend: The part of a website users see and interact with (HTML, CSS, JavaScript).
  • Backend: The server-side logic, databases, and APIs that power a website.

2. Web Communication Protocol (HTTP)

HTTP (Hypertext Transfer Protocol) is the foundation of communication on the web. It defines how messages are formatted and transmitted between web servers and browsers.

  • HTTPS: A secure version of HTTP using encryption (SSL/TLS).
  • Methods:
    • GET: Requests data from a server.
    • POST: Sends data to a server.
    • PUT: Updates existing data on a server.
    • DELETE: Removes data from a server.

3. Web Generations

  • Web 1.0 (Static Web): Early web pages, mostly read-only and static, with minimal interactivity.
  • Web 2.0 (Dynamic & Social Web): Interactive, user-generated content (social media, blogs), and dynamic websites.
  • Web 3.0 (Semantic Web): AI-driven, decentralized web focused on smarter data and personalization.

4. Standards & Constraints

  • W3C (World Wide Web Consortium): Defines web development standards like HTML, CSS, and JavaScript.
  • ECMA (European Computer Manufacturers Association): Standardizes JavaScript (ECMAScript).
  • Constraints:
    • Cross-Browser Compatibility: Websites should work the same on all browsers.
    • Responsive Design: Adapting web pages for different screen sizes (mobile, tablet, desktop).
    • Accessibility: Ensuring websites are usable by people with disabilities.
