Web Applications & Web Architectures

Understanding Different Types of Web Apps and How They Are Structured

1. Categories of Web Applications

  • Static Web Applications: Simple websites with fixed content, usually built with HTML and CSS only.
  • Dynamic Web Applications: Content changes based on user interaction, using server-side scripting (like PHP, Node.js) and databases.
  • Single-Page Applications (SPA): Entire website loads once, and only certain parts update without refreshing the page (React, Vue).
  • Multi-Page Applications (MPA): Traditional web apps where each action loads a new page from the server.
  • eCommerce Applications: Web apps for online shopping, featuring product catalogs, carts, and secure payments.
  • Portal Web Applications: Centralized platforms providing different services like emails, forums, and dashboards.

2. Web Architectures

  • Monolithic Architecture: A single, unified codebase handling both frontend and backend, easier to deploy but harder to scale.
  • Client-Server Architecture: The client (browser) requests data from the server, and the server responds with the needed resources.
  • Three-Tier Architecture: Divides the application into three layers â€” Presentation (UI), Logic (Backend), and Data (Database).
  • Microservices Architecture: Breaks the application into small, independent services, making it highly scalable and flexible.
  • Serverless Architecture: Backend logic runs on cloud-based infrastructure, where servers are managed by cloud providers (like AWS Lambda).
